HPC Guideline

EE HPC cluster is shared among multiple users, thus your actions can have serious impacts on the HPC system and affect the other users. The following policies and rules are in place to ensure a fair use of the computing resources and prevent unauthorized or malicious use.

EE Faculty and academic staff members, research staff, research students and FYP/Dissertation students are eligible users of EE HPC cluster.

To apply for a HPC account, please fill out the online form. Endorsement from a EE faculty member is needed. The maximum number of users under a faculty (including the faculty) is 4. All applications are subject to approval by the department.

Access to HPC clusters is to be via secure communication channel (e.g. ssh) to Login node. Compute nodes are intended for handling heavy computational work, and must be accessed via the resource management system (SLURM) only. Direct access to compute nodes is not permitted.

Master node and Login node are used for interactive work required to prepare computational tasks, as well as pre-processing and post-processing works such as source code editing, compilation, program testing, job submission and data analysis. All processes use heavy computing resources (CPU and memory) may be terminated without notice.

All computational jobs run on HPC cluster must be submitted via the resource management system (SLURM). This enable resources to be sensibly allocated and most efficiently used.

The ceiling for each user to use the HPC cluster concurrently (if more than one job under his/ her HPC account) is 112 CPU cores, 4 GPU cards continuously. The maximum time (walltime) of each job is 7 days. Request for more GPU cards will be considered on a case-by-case basis.

Each HPC user can submit 8 jobs in the queue, and run 4 jobs simultaneously.

The storage of the home directory for each user is 300 GB. Requests for more storage will be considered on a case-by-case basis.

For security reason, the HPC cluster can only be accessed within CityU campus network. CityU email addresses and EIDs should be used for identity verification purpose. To access EE HPC from outside of CityU campus, CSC VPN is recommended.

Users are suggested to check if a particular software has already been installed in HPC cluster before they submit a job request. If a user wants to apply for a new software to be installed, he/she should make sure that the software is a legal copy and sufficient licence(s) has/have been acquired in compliance with the licence agreement of the software.

Users should back up their data from time to time and CNS team are not liable to any data loss due to hardware problems and other unpredictable causes.

CNS team will plan and inform users of HPC cluster maintenance (via EE webpage or email announcement), however the cluster systems may require shutting down without prior notice in case of an emergency or unavoidable maintenance.
